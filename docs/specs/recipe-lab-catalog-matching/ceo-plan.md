# CEO Plan: Recipe Lab → Catalog Ingredient Matching

## Vision

### 10x Check
A **self-improving Catalog** where every chef interaction makes the system smarter. Every disambiguation becomes a permanent alias. Every approved pending request seeds the catalog. Over time, CulinAIre's catalog uniquely understands chef-narrative — a competitive moat that Toast and Marketman cannot replicate because their catalogs are static and they don't have a recipe-creation surface to build the feedback loop on.

### Platonic Ideal
```
Chef:   "Build me a pasta dish at $3.50 plate cost."
AI:     [3 variations appear. Each shows live Food Cost % vs. current prices.]
Chef:   picks one, taps "save to menu."
System: cost numbers already correct. No matching step. No modal. No triangles.

Next morning, Stock Room receives an order. Olive oil price up 12%.
System silently re-prices every affected menu item.
Owner's phone: "Aglio e Olio food cost rose from 24% to 27% due to olive oil.
                Tap to see what to do."
```

### Strategic framing
- **Wedge:** CulinAIre's wedge is solving food cost, profitability tracking, and inventory. The culinary world has had no real tech disruption in a decade. CulinAIre IS that disruption. Costing is not a feature — it IS the wedge.
- **Recipe Lab is the trust funnel.** Chefs are AI-skeptical. Recipe Lab exists to build trust through low-stakes creative play before chefs let the AI touch their P&L. The $0.00 cost screen is a trust-fund collapse, not a feature defect.
- **Apple north star.** Chefs are not tech-savvy. The product follows Apple's philosophy — simplify processes, let the wiring, backend, AI do the heavy lifting. The chef should not have to configure, select, resolve, or review. The system resolves; the chef overrides when they disagree.
- **Two entry points must complement each other.** Manual chef entry AND Recipe Lab import both lead to Menu Item ingredient rows. Same data model. Same matching pipeline. Same UX. The architectural fix lives at the API layer (match-bulk + resolver + escalation), so both flows benefit.

## Accepted Scope (added to this plan)

1. **Prompt-side split.** Edit Recipe Lab prompts (DB-managed) so LLM returns `canonicalName` + `descriptor` separately. Update `RecipeOutputSchema` Zod.
2. **Server-side normalizer.** New `packages/server/src/services/ingredientNormalizer.ts`. Strip qualifiers after `,` or `—`, lowercase, trim. Conservative. Eval test corpus.
3. **Shared resolver service.** New `packages/server/src/services/ingredientResolver.ts` + `useIngredientResolver` hook. Used by BOTH manual-entry and recipe-import paths.
4. **Silent best-guess linking.** When matchBulk returns ambiguous, auto-pick highest-scoring candidate. Subtle inline cue. No modal. Auto-write alias for self-improvement.
5. **Silent auto-escalation.** When matchBulk returns unmatched, silently create `pending_catalog_request`. Show estimated cost or "learning this ingredient" indicator. Never block save.
6. **Backfill script.** Idempotent. Runs `normalizeIngredientName` over every existing `recipe.recipe_data.ingredients[].name`. SKIP LOCKED for concurrency.
7. **EXPANSION #1: Live food-cost meter in Recipe Lab.** Plate cost builds in real time at the top of the page as the chef adds ingredients.
8. **Audit logs.** Auto-written aliases AND admin approvals of pending_catalog_request both logged for trust traceability.
9. **Feature flag rollout.** `enableSilentMatching` env var. 48h canary on named third-party chef tenant before full rollout.

## Critical gaps (MUST-FIX before ship)

1. **Prompt drift fallback heuristic.** When LLM returns old shape (no canonicalName field), split `name` on first comma as canonical. Log as "prompt drift" alert.
2. **Normalizer over-strip detection.** Eval suite covers "garlic powder" → "garlic" canary cases. CI fails if regression > 5%.
3. **DB pool exhaustion retry.** matchBulk caller retries 2x with exponential backoff before triggering "learning" escalation.
4. **Ambiguous-tied tiebreaker.** Deterministic rule: most-recently-used alias for this org wins; else lowest UUID for deterministic behavior.
